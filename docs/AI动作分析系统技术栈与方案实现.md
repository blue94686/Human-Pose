# 民体动作智能分析系统技术栈与方案实现

更新日期：2026-04-29

## 1. 目标与范围

本系统面向民体项目中的教学纠错、训练评估与科研实验，目标是建立一套本地化、可复核、可追溯的动作智能分析方案。

系统同时覆盖以下 4 类需求：

1. 教学纠错
2. 模板对比
3. 科研数据留存
4. Windows `.exe` 交付

## 2. 技术栈

### 2.1 桌面端

- `PySide6`
- 多线程 GUI 架构
- 三栏式主界面
- 状态栏 ETA
- 实时建议流

### 2.2 视觉与姿态

- `OpenCV`
- `YOLOv8-Pose`
- 光流兜底
- 骨架对比可视化

### 2.3 数值计算

- `NumPy`
- `SciPy`
- `pandas`
- Procrustes 对齐
- 差分 / 梯度导数

### 2.4 导出与中文报表

- `openpyxl`
- UTF-8 中文导出
- 结构化 Excel / CSV / JSON / Markdown

## 3. 总体架构

核心链路如下：

1. 读取视频帧
2. 姿态估计
3. 姿态缺失时启用光流兜底
4. 关键点稳定化与平滑
5. 提取躯干、双臂、平衡与节奏特征
6. 实时阶段推断
7. 实时问题识别
8. 实时建议生成
9. 视频结束后进行正式阶段划分
10. 执行模板对比
11. 计算科研指标
12. 导出结果文件

## 4. 姿态识别与兜底逻辑

### 4.1 正常路径

默认使用 `YOLOv8-Pose` 完成：

- 人体关键点识别
- 关键点置信度估计
- 跟踪模式标记
- 骨架可视化

### 4.2 兜底路径

当姿态后端不可用时，系统退化为光流兜底模式，可继续提供：

- 基础运动强度
- 左右运动平衡
- 候选阶段提示
- 有限的实时反馈

### 4.3 GUI 真实失败原因显示

GUI 不再只显示“当前为光流兜底模式，无骨架可视化”，而是直接展示真实原因，例如：

- 未安装 `Ultralytics`
- 权重文件不存在：`xxx.pt`
- 姿态模型加载失败：`...`
- 姿态推理失败：`...`

这些信息会同步显示在：

- 左侧姿态提示区
- 底部状态栏
- 中央实时状态区
- 叠加预览提示

## 6. 核心动作特征

### 6.1 躯干倾斜

- 通过肩中点与髋中点构建主轴
- 计算主轴相对垂直方向的偏移角度

### 6.2 双臂对称误差

- 综合左右抬臂角度
- 综合左右腕部高度
- 综合肩部平衡程度

### 6.3 模板距离

- 基于关键点位置与模板基线对齐后计算偏差
- 每帧结果写入 `FrameMetrics.template_distance`

## 7. 实时问题识别与建议引擎

### 7.1 当前主要阈值

- 躯干倾斜提示阈值：`15°`
- 双臂对称误差提示阈值：`10`
- 模板距离提示阈值：`0.5`
- 微动速度阈值：`5 px/帧`

### 7.2 建议规则映射

#### 躯干倾斜

条件：

```text
trunk_angle > 15°
```

建议：

```text
姿态偏差：躯干明显倾斜。建议：百会穴上领，收腹提肛，保持脊柱中正。
```

#### 双臂不对称

条件：

```text
arm_symmetry_error > 10
```

建议：

```text
动作非对称：双臂高度不一。建议：平衡双肩发力，招式需匀称舒展。
```

#### 框架偏差

条件：

```text
template_distance > 0.5
```

建议：

```text
框架偏差：肢体伸展不足。建议：拉伸四肢，定势时注意沉肩坠肘。
```

#### 控制力不足

条件：

- 关节抖动偏大
- 稳定度偏低

建议：

```text
控制力不足：发现肢体细微抖动。建议：减慢速度，加强核心支撑，体验气沉丹田。
```

## 8. 同源视频 100 分修正机制

### 8.1 触发条件

若满足以下任一条件：

- 测试视频路径与模板源路径一致
- 平均模板距离 `< 1e-3`
- 平均模板距离极低并满足极高相似度兜底

### 8.2 覆盖动作

系统会强制：

- 综合得分 `100.0`
- 分项得分 `100.0`
- 偏差项 `0.0`
- 总结文本追加：
  `检测到基准视频自校准，动作完美匹配。`

## 9. 科研指标实现细节

### 9.1 已有 5 项抑制类指标

1. 肢体晃动度
2. 躯干稳定度
3. 规范偏差
4. 角度稳定性
5. 无效小动作频次

### 9.2 身体稳定性参数 `Body Stability CV`

输入：

- `neck_pos`
- `hip_center_pos`

实现口径：

- 计算每帧“颈-髋连线”相对垂直轴的偏移角度
- 形成角度序列
- 计算：

```text
CV = std(angle_series) / max(abs(mean(angle_series)), epsilon)
```

实现要点：

- COCO 无显式 `neck` 点时，以左右肩中点作为颈点代理
- 使用 `np.nan_to_num`
- 空值可插值

### 9.3 动作流畅性参数 `Action Fluency`

输入：

- `hip_center_pos`
- `action_stage`

实现口径：

- 仅取 `过渡` 阶段
- 对髋部中心轨迹求：
  - 速度
  - 加速度
  - 急动度
- 导出：
  - `Transition_Acc_Peak`
  - `Transition_Jerk_Mean`

### 9.4 身体调控能力参数 `Body Control Ratio`

输入：

- `hip_center_pos`
- `action_stage`

实现口径：

- 对每个连续 `定势` 片段，截取末尾 10 帧
- 对每个连续 `过渡` 片段，截取中间 10 帧
- 分别计算位移幅度
- 比值公式：

```text
Ratio = Σ过渡期位移幅度 / Σ定势期位移幅度
```

实现要点：

- 若片段长度不足 10 帧，则按实际帧数计算
- 若正式 `定势` / `过渡` 缺失，则回退到候选阶段
- 若没有可用片段，则返回 `0.0`

### 9.5 容错要求

已实现以下防御：

- `np.nan_to_num`
- `NaN` / `inf` 过滤
- 阶段长度不足防御
- 候选阶段回退
